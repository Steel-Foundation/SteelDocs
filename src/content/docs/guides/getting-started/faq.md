Here you will find answers to most of the common questions!
DISCLAIMER: Steel is pre-alpha, so things and answers can change frequently!

## How do I install Steel?
You can find the full guide [here](../running).

## Is there an Egg?
Currently, there is no egg (*.egg) for a provider to host Steel. Also, Steel is currently in pre-alpha, so it shouldn't be used on a server.

## Where is the jar (*.jar) file?
Steel is not written in Java, so it doesn't have a jar (*.jar). You can only find a Windows, Linux, or Mac executable on our homepage or GitHub repo.

## Will Steel have vanilla parity?
Short and simple: that is the main goal.

## Does Steel have plugins, and will my Paper/Bukkit plugin work on Steel?
Currently not. Steel is a WIP, and plugins are planned but not yet possible. Steel will have its own API, which will not be like the Paper or Bukkit API!

## In which language will plugins be written?
The main API will be in Rust, but the community can add support for WASM or other technologies to enable more languages.

## Does Steel have mods, and can I use my current ones?
NeoForge/Forge/Fabric mods will not be supported. As with plugins, Steel will use Rust, and the internal functionality differs, as does what will ultimately be possible.

## I have a problem, what should I do?
You can report it on [GitHub](https://github.com/Steel-Foundation/SteelMC/issues), or join our [Discord](https://discord.gg/MwChEHnAbh) and post it in the bugs channel.
